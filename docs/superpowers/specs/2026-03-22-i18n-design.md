# i18n (EN/DE) Design Spec

## Goal
Add German/English language toggle to the PRESENCE Protocol app — client-side only, no routing changes, persisted in `localStorage`.

## Architecture

**New files:**
- `web/src/lib/i18n.ts` — typed translations object (`en` + `de`) for all strings across all pages, plus `t(lang, key)` helper
- `web/src/context/LanguageContext.tsx` — React context exposing `{ lang, setLang, t }`. Reads `localStorage('lang')` on mount, defaults to `'en'`

**Modified files:**
- `web/src/app/layout.tsx` — wrap `{children}` in `<LanguageProvider>`. **Do NOT add `'use client'` to `layout.tsx`** — it must remain a Server Component to keep `export const metadata` and `next/font`. `LanguageProvider` itself carries `'use client'`; wrapping server component children with a client component is the correct App Router pattern.
- `web/src/app/components/NavBar.tsx` — add `DE | EN` toggle button (monospace, right side)
- `web/src/app/page.tsx` — replace hardcoded strings with `t(key)`
- `web/src/app/call/page.tsx` — same
- `web/src/app/phone/page.tsx` — same
- `web/src/app/handshake/page.tsx` — same
- `web/src/app/demo/page.tsx` — same
- `web/src/app/demo/BankPanel.tsx` — same
- `web/src/app/demo/PhonePanel.tsx` — same
- `web/src/app/demo/HandshakePanel.tsx` — same

## Language Toggle UI
- Small monospace `[DE]` / `[EN]` button in the NavBar right side
- When English active: shows `[DE]` (click to switch to German)
- When German active: shows `[EN]` (click to switch to English)
- Color: `#555` default, `#2563eb` on hover
- No flag icons — code only

## Translations

All keys are flat strings. `t` is a **closure** — it captures the active language from context state, so callers only pass the key:
```ts
// Context exposes:
t: (key: keyof typeof translations.en) => string
// Usage anywhere in the tree:
const { t } = useLanguage()
t('home.tagline') // returns 'Trust beyond the voice' or 'Vertrauen über Stimme'
```

The `t(lang, key)` form mentioned in the architecture section above is informal shorthand — the actual implementation uses the single-argument closure form shown here.

## Behaviour
- Default language: `en`
- On toggle: update context state + write `localStorage.setItem('lang', newLang)`
- SSR/hydration safe: `LanguageContext` initialises state to `'en'` synchronously (for server render), then reads `localStorage.getItem('lang')` inside a `useEffect` on mount and updates state if `'de'` is stored. This prevents hydration mismatches.
- No page reload, no routing change — purely reactive via context

## Implementation notes
- `PHASE_TEXT` in `demo/PhonePanel.tsx` and status strings in `call/page.tsx` are module-level constants — they must be moved inside the component body so they can call `t()` from context.
- `NAV_LINKS` labels in `NavBar.tsx` are also static — translate them inline inside the component using `t()` rather than a static array.
